## The 14-Column Standard

Every CSV imported into the WASP system MUST follow this exact structure:

```csv
name,address,city,postal_code,email,email_secondary,phone,website,lat,lng,facebook_url,instagram,contact_person,extra_details
```

### Why This Matrix?

1. **CONSISTENCY** - All countries, all imports use identical columns
2. **FLEXIBILITY** - Empty fields are OK; optional fields can be blank
3. **SCALABILITY** - Single schema handles 1K to 1M+ records identically
4. **FUTURE-PROOF** - No need to add new columns later (they're all here)
5. **SEARCH** - Full-text search, filters, and queries work across all data

## Column Reference

### Identity & Organization
| Column | Required? | Purpose | Example |
|--------|-----------|---------|---------|
| **name** | ✅ YES | Organization's full legal name | "PETA Inc", "Animal Rescue UK", "Wildlife Trust" |

### Location
| Column | Required? | Purpose | Example |
|--------|-----------|---------|---------|
| address | optional | Physical street address | "123 Main St", "5 Oxford Road" |
| city | optional | City/town/suburb | "Rome", "London", "Sydney", "New York" |
| postal_code | optional | ZIP/postal code | "28100", "EC1A 1BB", "2000" |
| lat | optional | Latitude (with lng) | "41.8719", "-35.0966" |
| lng | optional | Longitude (with lat) | "12.5674", "173.2548" |

### Contact Information
| Column | Required? | Purpose | Example |
|--------|-----------|---------|---------|
| email | optional | Primary email address | "info@org.com", "contact@org.org" |
| email_secondary | optional | Secondary/backup email | "support@org.com", "admin@org.org" |
| phone | optional | Phone number (any format) | "+39 06 123 4567", "001-202-555-0173" |
| contact_person | optional | Name of primary contact | "Jane Smith", "Dr. John Doe" |

### Digital Presence
| Column | Required? | Purpose | Example |
|--------|-----------|---------|---------|
| website | optional | Organization website URL | "https://example.org", "www.org.com" |
| facebook_url | optional | Facebook page/profile URL | "https://facebook.com/orgname" |
| instagram | optional | Instagram handle or URL | "@orgname", "https://instagram.com/orgname" |

### Notes & Metadata
| Column | Required? | Purpose | Example |
|--------|-----------|---------|---------|
| extra_details | optional | Any additional information | "Founded 1995", "Focuses on endangered species" |

## What Gets Generated (NOT in CSV)

These fields are ADDED by the system - do NOT include them in your CSV:

- **code** - Auto-generated unique code (ITA0001, CAN0010, NZL0010, etc.)
- **country** - Auto-assigned from import script (not user input)
- **id** - UUID, auto-generated by Supabase
- **created_at** - Timestamp when record is inserted
- **updated_at** - Timestamp when record is updated
- **search_vector** - Full-text search index (auto-maintained)

## Validation Rules (STRICT)

### MUST HAVE:
- ✅ `name` field = non-empty

### NICE TO HAVE:
- 📍 Coordinates (`lat` + `lng` both present, valid decimal numbers)
- 📧 Email address (valid format)
- 🌐 Website URL
- 📱 Phone number
- 👤 Contact person name

### OPTIONAL BUT STANDARDIZED:
- Everything else (all 14 columns defined; missing = blank/null)

### WILL REJECT:
- ❌ Empty name field
- ❌ One coordinate without the other (lat without lng, or vice versa)
- ❌ Invalid coordinate format (not a number, or out of range)
- ❌ Invalid email format (if email field is populated)

## Import Process (Same for ALL Countries)

1. **Prepare CSV** with 14-column standard (can use any variations above)
2. **Create import script** from `import_country_template.js` template
3. **Configure** country name + CSV path + optional column mapping
4. **Run import** - system validates, generates codes, inserts to database
5. **Verify** - audit report shows success/warnings
6. **Done** - no manual cleanup needed

## Examples: How Different CSVs Map to Standard

### Example 1: Canada (already has good headers)
```
name,address,website,email,lat,lng,phone,facebook_url,instagram
Alberta Animal Rescue,...
```
→ Maps directly (instagram is recognized as a standard field)

### Example 2: New Zealand (needs slight mapping)
```
NAME,CITY,ADDRESS,WEB,EMAIL,LAT,LON,TELEFONO,FACEBOOK
SPCA NZ,...
```
→ System recognizes: NAME→name, LAT→lat, LON→lng, WEB→website, TELEFONO→phone, FACEBOOK→facebook_url, etc.

### Example 3: Italy (will need custom mapping in script)
```
Codice,Ragione Sociale,Indirizzo,Città,Email,Telefono,Sito Web
ITA0001,WWF Italia,...
```
→ Script defines: `columnMapping: { "Ragione Sociale": "name", "Indirizzo": "address", "Città": "city", "Sito Web": "website" }`
