# WASP Association Import Schema & Process

## Universal Schema Definition

**All countries use the EXACT SAME 14-column CSV format. No exceptions.**

### Standardized CSV Header (in this exact order)
```
name,address,city,postal_code,email,email_secondary,phone,website,lat,lng,facebook_url,instagram,contact_person,extra_details
```

### Column Definitions & Rules

| Column | Required? | Type | Examples | Notes |
|--------|-----------|------|----------|-------|
| `name` | **YES** | text | "PETA", "Animal Rescue UK" | Association full legal name. ONLY truly required field. |
| `address` | optional | text | "123 Main St", "5 Oxford Road" | Street address or physical location. |
| `city` | optional | text | "Rome", "London", "Sydney" | City/town/suburb name. |
| `postal_code` | optional | text | "28100", "EC1A 1BB" | ZIP/postal code. |
| `email` | optional | text | "info@org.com" | Primary contact email. Validated format. |
| `email_secondary` | optional | text | "contact@org.com" | Secondary email if available. Validated format. |
| `phone` | optional | text | "+39 06 123 4567" | Phone number in any format. |
| `website` | optional | text | "https://example.org/" | Organization website URL. |
| `lat` | optional | decimal | 41.8719, -35.0966 | Latitude coordinate. Must be paired with `lng`. |
| `lng` | optional | decimal | 12.5674, 173.2548 | Longitude coordinate. Must be paired with `lat`. |
| `facebook_url` | optional | text | "https://facebook.com/org" | Facebook profile URL. |
| `instagram` | optional | text | "@orgname" or URL | Instagram handle or profile URL. |
| `contact_person` | optional | text | "Jane Doe" | Name of primary contact/representative. |
| `extra_details` | optional | text | "Founded 1995, focuses on..." | Any additional notes or description. |

### Auto-Generated Fields (DO NOT include in CSV)
These are added by the import system automatically:
- `code` - Generated automatically (e.g., ITA0001, CAN0010, NZL0010)
- `country` - Set from import script configuration
- `id` - UUID, auto-generated by Supabase
- `search_vector` - Full-text search index, auto-maintained
- `created_at` - Auto-set to import timestamp
- `updated_at` - Auto-set to import timestamp

## Import Process (Universal for ALL countries)

### Step 1: Data Preparation
```
INPUT: country_data.csv with 14-column standardized headers
PROCESS:
  1. Detect encoding (UTF-8 or latin1)
  2. Parse CSV with header row
  3. Normalize column names to schema (lowercase, map source variations → standard 14 columns)
  4. Validate ONLY required field present: name
  5. Clean data (trim whitespace, remove duplicates, handle null values)
  6. Default missing optional fields to empty string or null
OUTPUT: Normalized records array
```

### Step 2: Validation
```
FOR EACH RECORD:
  ✓ name: MUST be non-empty (ONLY truly required field)
  ✓ lat/lng: If one present, both must be valid decimal numbers (-90 to 90, -180 to 180)
  ✓ email: If present, must be valid email format (basic check)
  ✓ email_secondary: If present, must be valid email format
  ✓ phone: If present, must not be empty
  
Optional fields (address, city, postal_code, website, etc.) = if missing, default to empty string
  
REJECT: Records with empty name
LOG: Invalid records to audit file
```

### Step 3: Deduplication
```
FOR EACH RECORD:
  1. Check if code already exists in database
  2. If exists: SKIP (log as duplicate)
  3. If new: PROCEED to batch insert
```

### Step 4: Batch Insert
```
BATCH SIZE: 100 records per insert
  1. Create insert batch
  2. Execute insert via Supabase
  3. Log success count
  4. Move to next batch
  5. Repeat until all records processed
```

### Step 5: Verification
```
POST-IMPORT:
  1. Count total records inserted
  2. Count records with coordinates (lat/lng present)
  3. Count records without coordinates (missing data)
  4. Verify no duplicates introduced
  5. Generate audit report
```

## File Naming Convention

All country imports follow this pattern:

```
import_[COUNTRY_LOWERCASE].js
Example: import_canada.js, import_usa.js, import_australia.js
```

Each script:
1. Takes source CSV path as argument
2. Uses universal validation/import logic
3. Logs results to `[country]_import_audit.log`
4. Exports cleaned data to `[country]_associations_export.csv` (for verification)

## Error Handling

**Non-fatal errors (skip record, log, continue):**
- Missing optional fields
- Invalid coordinates
- Invalid email format
- Duplicate code

**Fatal errors (stop import, report):**
- Missing required fields (code, name, country, city, address)
- Invalid country name
- CSV parse error
- Database connection error

## Verification Queries

After each import, run:

```sql
-- Total records by country (with coordinate coverage)
SELECT country, COUNT(*) as total, 
  COUNT(CASE WHEN lat IS NOT NULL AND lng IS NOT NULL THEN 1 END) as with_coords,
  COUNT(CASE WHEN lat IS NOT NULL AND lng IS NOT NULL THEN 1 END) * 100.0 / COUNT(*) as coord_coverage_pct
FROM associations
GROUP BY country
ORDER BY country;

-- Check for duplicate codes (should be 0)
SELECT code, COUNT(*) as cnt
FROM associations
GROUP BY code 
HAVING COUNT(*) > 1
ORDER BY cnt DESC;

-- Data completeness check
SELECT country, 
  COUNT(CASE WHEN email IS NULL OR email = '' THEN 1 END) as no_email,
  COUNT(CASE WHEN website IS NULL OR website = '' THEN 1 END) as no_website,
  COUNT(CASE WHEN phone IS NULL OR phone = '' THEN 1 END) as no_phone,
  COUNT(CASE WHEN address IS NULL OR address = '' THEN 1 END) as no_address
FROM associations
GROUP BY country
ORDER BY country;
```

## Summary: The Contract

**Every country import MUST:**
1. ✓ Use identical 14-column CSV format (name, address, city, postal_code, email, email_secondary, phone, website, lat, lng, facebook_url, instagram, contact_person, extra_details)
2. ✓ Validate ONLY required field: name (everything else optional)
3. ✓ Auto-generate codes (never import codes; system creates them)
4. ✓ Auto-assign country (from script configuration)
5. ✓ Use batch insert pattern (100 records per batch)
6. ✓ Run verification checks post-import
7. ✓ Generate audit log with dates, counts, warnings
8. ✓ Export normalized CSV for inspection & record-keeping

**NO EXCEPTIONS. NO VARIATIONS. NO CUSTOM LOGIC PER COUNTRY.**

This is the foundation for handling 1M+ records consistently across all countries.
